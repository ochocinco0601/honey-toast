# Start Here

The Observability Body of Knowledge (O-BOK) is an opinionated map of the observability domain, organized around a specific methodology: **business observability** — defining success in business health terms before designing signals, dashboards, or alerts. Current working version — iterates with use.

|---|---|

| **See** what questions this BOK is useful for | [Questions the BOK Answers](questions-the-bok-answers.md) — the full set of questions it should answer, and explicit scope boundaries |

## Producing from this BOK

This BOK is a **layer-5 substrate** — the depth other work pulls from, not a form to hand a consumer as-is. When you produce for a real audience, slot the ask by consumer layer and cut the fitted form. See the companion [The Practice System](../the-practice-system.md).

## What this is not

Not a playbook. Not canonical. Not a finished product. Content states on every Knowledge Area page are honest about what's filled in.
